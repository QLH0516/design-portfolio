## 二、视觉设计系统

### 2.1 色彩体系

| 角色 | 色彩 |
|------|------|
| 主色 | 黑色 #0A0A0A、白色 #FAFAFA |
| 强调色 | 深红色 #C92A2A（悬停变亮 #E03E3E） |
| 中性色 | 浅灰 #F5F5F5、中灰 #999999、深灰 #333333 |

**配色策略**: 采用极致的黑白对比作为基底，以单一强调色（红色）作为视线引导和交互反馈的手段。整体配色克制，突出内容本身而非装饰。

**使用规则**:
- 强调色仅用于：链接文本、hover 状态、关键 CTA、滚动指示器
- 中性色用于：分割线、次要文本、背景层次区分
- 全黑背景（#0A0A0A）仅用于 Hero 全屏区或项目详情沉浸区
- 避免在纯白背景上使用浅灰文字（对比度不足 WCAG AA）

### 2.2 排版系统

| 属性 | 规则 |
|------|------|
| 主要字体 | **Neue Haas Grotesk**（首选）或 **Helvetica Now Text / Display** |
| 标题层级 | h1: clamp(3.5rem, 10vw, 7rem)，字重 700-800，字母间距 -0.02em |
| 次级标题 | h2: clamp(2rem, 5vw, 4rem)，字重 600，字母间距 0 |
| 正文层级 | clamp(1rem, 1.5vw, 1.25rem)，字重 350-400，行高 1.7 |
| 辅助文本 | 0.75rem，大写 + 字母间距 0.15em，字重 500 |
| 对齐方式 | 左对齐为主；引语/页脚区域可居中对齐 |
| 字体基调 | 干净、现代、有建筑感的几何字体，传达专业与信赖感 |

**排版策略**: 通过极端的大小对比制造视觉冲击力。大量留白让每个文字都有呼吸空间。标题通常占据页面主导地位。正文较细较轻，形成与粗标题的张力。

**实际实现建议**:
- 如果无法获取 Neue Haas Grotesk 许可，备选方案按优先级排序：
  1. **Inter Display**（免费，可自托管）
  2. **SF Pro Display**（macOS 系统字体备选）
  3. **Manrope**（开源，几何感强）
- 必须自托管 WOFF2，禁用 Google Fonts CDN
- 推荐加载 3 个字重：Light (300)、Regular (400)、Bold (700)

### 2.3 网格与布局

- **布局类型**: 非对称网格 + 全出血（full-bleed）区块
- **栅格系统**: 12列灵活网格，有时刻意打破网格制造动感
- **内容容器**: 主内容区 max-width: 1200px，居中；全出血元素无视容器边界
- **留白**: 大量使用垂直留白（section 间距通常在 120-200px）；内边距慷慨
- **Z轴层次**: 通过叠加、视差滚动、固定定位创造深度
- **响应式**: 桌面优先，平板和移动端保持克制精简的布局，字体逐级缩小
- **特色布局模式**:
  - 项目网格：2列（桌面）/ 1列（移动），卡片间大间距
  - 图文并置：文字块与图片块左右交错排列（offset 布局）
  - 全出血图片：图片突破容器边界延伸到屏幕边缘

### 2.4 图像与媒体处理规范

| 属性 | 规范 |
|------|------|
| 图像风格 | 黑白或高饱和度摄影；建筑感构图；人物与环境的关系记录 |
| 图像处理 | 高对比度后期，经常使用颗粒状黑白胶片质感 |
| 视频 | 全屏自动播放背景视频、循环短片；克制且具有氛围感 |
| 图文关系 | 图片与文字呈并置或交错排列，形成对话关系 |
| 图标 | 极少使用图标库；如有则采用自绘极简线条图标 |
| 占位策略 | 真实资产就绪前使用低饱和度纯色块 + 几何形状组合占位 |

## 三、交互与动效设计

| 交互类型 | 描述 |
|----------|------|
| 页面过渡 | 平滑无缝的页面切换，无闪烁白屏（SPA 或 PJAX 技术） |
| 滚动触发 | 元素随滚动淡入、上移（fade-in + translateY），节奏舒缓 |
| Hover 效果 | 链接下划线动画、图片轻微缩放、色彩转换（如黑白变彩色） |
| 光标交互 | 自定义光标 / 光标跟随效果（圆形光环跟随鼠标） |
| 菜单动画 | 全屏覆盖式菜单，展开/收起有平滑缓动 cubic-bezier(0.16, 1, 0.3, 1) |
| 加载状态 | 极简加载指示器（纤细线形或点状），过渡优雅 |
| 视差滚动 | 背景与前景以不同速度移动，增加沉浸感（通常 0.5x 速度差） |
| 图片揭示 | 图片在滚动进入时采用 clip-path 遮罩揭示动画 |
| 滚动静音 | 页面间导航时暂停所有动画，避免性能浪费 |

**动效哲学**: 动效不是炫技，而是为叙事服务。每一个动效都缓慢、优雅且有目的，强调内容之间的逻辑关系和节奏感。所有动效持续时间应在 0.6s-1.2s 之间，缓动函数统一。

**性能约束**:
- 不使用 jQuery 或重量级动画库
- 优先使用 CSS will-change 和 transform/opacity 动画（GPU 加速）
- GSAP 仅用于复杂时间线动画，简单滚动触发用 Intersection Observer
- requestAnimationFrame 驱动的动画必须搭配 visibility change 暂停逻辑

## 四、信息架构

### 4.1 路由结构（仿 studionamma.com）

| 路由 | 页面 | 布局特点 |
|------|------|----------|
| / | Home | Hero 全屏 + 精选项目网格 + 工作室简介入口 |
| /work | 项目列表 | 项目网格展示，hover 显示标题和类别 |
| /work/:slug | 项目详情 | 沉浸式叙事布局，图文交错 |
| /about | 关于我们 | 工作室理念 + 团队 + 客户名录 |
| /contact | 联系方式 | 极简联系表单 + 社交链接 |

### 4.2 页面布局模板

**标准页面布局（由上至下）**:

1. 固定导航栏 — 左侧 Logo，右侧 Menu 按钮（汉堡图标），透明背景
2. 全屏 Hero — 大标题 + 滚动指示器 + 可选背景视频
3. 内容区 — 2-3 个 section，每 section 间距 120-200px
4. 项目展示 — 2列网格，每项包含封面图 + 标题 + 类别标签
5. CTA / 页尾 — 简洁页脚，包含 Email / 社交 / 版权

**项目详情页布局**:

1. 项目 Hero（大标题 + 封面图 + 元数据）
2. 项目描述（左侧正文 + 右侧关键信息）
3. 图片画廊（全出血大图与居中文字块交错）
4. 过程展示（Before/After 或时间线）
5. 下一个项目导航 + 返回按钮

## 五、技术栈推测

| 层面 | 技术选择 |

## 六、设计原则总结

1. **克制即力量** — 用最少的设计元素传达最强的信息
2. **内容为王** — 设计服务于内容，从不喧宾夺主
3. **节奏与呼吸** — 慷慨的留白和从容的动效创造愉悦的浏览节奏
4. **一致的视觉语言** — 从色彩到排版到图像风格皆保持高度统一
5. **叙事驱动** — 每个页面都在讲述一个故事，用户被引导而非被推销
6. **手工感与数字融合** — 在数字媒介中保留手工设计的温度和细节
7. **细节决定品质** — 微交互动效、字距调整、对齐精度处处体现专业度
8. **性能即设计** — 加载速度和交互流畅度是用户体验不可分割的部分
9. **无障碍优先** — 高对比度保障可读性，语义化 HTML 保障屏幕阅读器兼容
10. **永恒而非时髦** — 设计不受短期趋势影响，追求经得起时间考验的经典感

## 七、具体页面逐区块参照

### 7.1 首页 Hero 区

- 全屏视口高度（100vh），居中布局
- 主标题占页面高度 40-50%，字重 700-800
- 标题下方 2-3 行描述性副文本（字号 1.25rem，字重 350）
- 右下角或底部有向下滚动指示器（动画箭头或细线）
- 背景：纯色（白/黑）或自动播放的静音视频循环
- 导航栏在此区域为透明（文字白色或黑色取决于背景）

### 7.2 项目网格区

- 2列网格，卡片间 gap 约 3-4rem
- 每张卡片：全出血图片 + 标题（左对齐）+ 类别标签（小号大写）
- Hover 效果：图片轻微放大（scale 1.02-1.05）+ 标题变强调色
- 交替行偏移：奇偶行的卡片起始位置微调，打破对齐的单调

### 7.3 关于区

- 工作室描述（左侧 60% 宽度）+ 关键数据（右侧 40%）
- 团队照片：黑白风格，排列松散
- 客户 Logo 列表：灰度，单行排列

### 7.4 联系/页脚

- 超大字号 Email 链接（clamp(2rem, 4vw, 3.5rem)）
- 社交链接：小号图标或纯文字链接
- 版权信息：极小字号的辅助文本
